### 이중포인터란?

- **포인터의 주소**를 저장하는 변수
- 선언 방법 : `자료형** 포인터이름;`

```c
//이중 포인터 사용방법
int x = 20;
int* p = &x;
int** pp = &p;
**pp = 50;
printf("change 함수 호출한 후의 x : %d\n", x);

//출력 : change 함수 호출한 후의 x : 50
```

- 사용하는 이유?
    
    : 값을 직접 넘겨주는 것 보다 주소값을 넘겨줌으로써 **시간적, 리소스적** 장점이 있다.
    
      ex) git
    

### 포인터를 사용하는 이유? == 객체 변수에 왜 레퍼런스를 저장하나요?

- 메모리에 직접 접근이 가능하기 때문에 주소를 참조해서 다양한 자료형 변수들의 접근과 조작이 편리하다.
- 데이터들을 공유할 수 있고, 이를 통해 **공간을 효율적으로 사용**할 수 있다.

`workspace` : 현재 작업중인 장소

`index(=stage)` : workspace의 수정된 소스를 저장하는 장소

`local repository` : commit까지의 저장소

`remote repository` : 원격 저장소,  clone을 통해 workspace로 가져올 수 있음

출처 : osteele.com

### git fetch랑 pull 차이

- fetch
    - remote repository의 커밋들을 local repository로 가져온다.
    - 자동 merge가 아니기 때문에 본인이 직접 확인 후 merge 해줘야함
- pull
    - remote repository의 정보를 가져오면서 local branch에 병합까지 수행해주는 명령어
    - conflict가 발생할 수 있음

### git은 commit 할때 파일내용을 sha1로 암호화

→ 새로 추가한 파일이 이전 commit된 파일과 동일한 내용일때 object에 파일을 추가하지 않고 .git/index에 파일명 저장(용량을 적게 관리하기 위해)

object 파일에서 리눅스 최대 파일 저장개수를 넘어서면 느려지기 때문에 prefix로 디렉토리(=폴더)를 구분

### git conflict

1. 충돌이란?
    
    두개의 파일을 병합하는 과정에서 생기는 문제
    
    파일이 같고 **수정한 부분 또한 같은 경우** 발생
    
2. conflict된 파일 내용
    
    ```
    <<<<<HEAD(내가 건드린 내용)
    
    a.text 내용????
    
    =======
    
    a.text 내용!!!!!
    
    >>>>>>41fe5f203djfk(원격지 최신 작업본 내용)
    ```
    

1. conflict 해결 방법
    
    : 충돌이 일어난 파일 수정 후 master에 merge!
